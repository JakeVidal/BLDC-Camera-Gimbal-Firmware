Problem Definition

What is a Gimbal?

A gimbal is a mechanical assembly with multiple axes of rotation, which allows an object placed at the centre to remain stationary as the frame moves freely. Gimbals find applications in many domains where stabilization is necessary.

Image capture, as an example, requires the compensation of vibration and movement to reduce image blurring during capture, and gimbals have often been used for this purpose.

Uvic AERO's Problem

AERO wanted our team to design the hardware and firmware of an electronic control system for an existing Brushless DC (BLDC) electric motor camera gimbal, with the goal of solving their camera stabilization problem.

Some of the capabilities and characteristics include:

  • Stabilizing at the pitch and yaw of a payload consisting of a camera
  • Smoothing movement to eliminate jitter from the image acquisition system
  • Receiving and obeying pan and tilt commands at a distance
  • Autonomous tracking of objects

Hardware Architecture

The hardware system includes two BLDC motors, an IMU breakout board, two motor encoders, and a custom PCB to interface them all. The PCB was required due to the lack of reliable and cost effective third party gimbal control boards.

The PCB houses an STM32F303 microcontroller and three DRV8313 motor drivers used to control the gimbal, allowing a three-axis gimbal in the future. The PCB includes three voltage regulators.The first is a 12V-8A buck converter that provides power to three motor drivers. The second rail is a 5V-5A buck converter that provides power to RC radio receivers and a Raspberry Pi 3B+ (used for object tracking). The third is a 3.3V-500mA LDO that is derived from the 5V rail, and used to power the STM32F303 and the I2C peripherals.

Firmware Architecture

We decided to use the FreeRTOS real-time operating system framework to structure and implement the embedded firmware project. The framework is easy to learn and brings quicker development times and higher concurrency safety to real-time systems. The gimbal is a system that must react to events with minimal latency which qualifies it as a real-time system that can benefit from FreeRTOS.

The diagram to the left shows the concurrent programming message passing model, which lends itself to the use of a real-time operating system.

Object Tracking Architecture

The object tracking system automatically directs the gimbal to follow a set of features that is at the center of the camera. To accomplish this, the team chose to separate the object tracking into its own computer, and for the purposes of this project, we used a Raspberry Pi.

The program works by first receiving an image from the camera, then passing the image through the image processing module, which contains the object tracking logic. Commands are sent to the gimbal's control system via the communications interface.
